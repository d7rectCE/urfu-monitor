# УрФУ — монитор конкурсных списков

Небольшое десктопное приложение на Python: следит за конкурсными списками
УрФУ по регистрационному номеру абитуриента и присылает системное уведомление,
когда что-то меняется — появились баллы за вступительное испытание, сменился
статус заявления, добавились индивидуальные достижения.

## Возможности

- все заявления по номеру в одной таблице: приоритет, конкурс, основа,
  институт, образовательная программа, направление, сумма баллов, ИД,
  оценки за каждое ВИ, статус, наличие оригинала документа;
- подсветка: зелёный — участвует в конкурсе, серый — на рассмотрении,
  жёлтый — строка изменилась с прошлой проверки;
- уведомления только на реальные изменения, с расшифровкой «было → стало»;
- отслеживание времени последней выгрузки данных вузом (`last_import`);
- сохранение настроек и автозапуск мониторинга;
- экспорт текущего снимка в JSON.

## Установка

```bash
git clone https://github.com/d7rectCE/urfu-monitor.git
cd urfu-monitor
pip install -r requirements.txt
python urfu_monitor.py
```

Нужен Python 3.9+. Tkinter входит в стандартную поставку Python на Windows и
macOS; на Linux может потребоваться отдельный пакет:

```bash
sudo apt install python3-tk
```

## Использование

Введите регистрационный номер (он указан в личном кабинете абитуриента),
задайте интервал проверки и нажмите «Старт». Настройки сохраняются в
`~/.urfu_monitor.json` и подставляются при следующем запуске.

Интервал по умолчанию — 180 секунд. Ставить меньше смысла нет: вуз
обновляет выгрузку заметно реже, а лишние запросы только нагружают сервер
приёмной комиссии.

## Как это работает

Приложение обращается к тому же HTTP-эндпоинту, который использует
официальная страница со списками:

```
GET https://urfu.ru/api/entrant/?page=1&size=10&search=<регистрационный номер>
```

Данные публичные — те же самые, что видит любой посетитель сайта. Персональных
сведений там нет, только регистрационные номера.

## Оговорки

Проект никак не связан с УрФУ и не является официальным. Эндпоинт нигде не
задокументирован, поэтому вуз может изменить или закрыть его в любой момент —
тогда приложение перестанет работать до обновления кода.

Приложение только читает открытые данные и делает это не чаще, чем человек,
вручную обновляющий страницу. Не увеличивайте частоту опроса и не используйте
код для массовой выгрузки всех списков целиком.

Решения о поступлении принимайте по официальным источникам — личному кабинету
абитуриента и приказам о зачислении, а не по этому монитору.
